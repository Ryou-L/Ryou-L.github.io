操作系统–文件系统

Abstract

  • 文件管理系统
  • 文件的逻辑组织
  • 文件目录
  • 文件共享
  • 文件存储空间的管理

文件管理系统的功能

File Management(文件管理)

  • File management system is considered part of the operating system.（文件管理系统被认为是操作系统的一部分。）
  • Input to applications is by means of a file.（应用程序的输入是通过文件的方式实现的。以文件为单位）
  • Output is saved in a file for long-term storage.（输出被保存在一个文件中以便长期存储）

Obiectives for a File Management System（文件管理系统的目标）

  • Meet the data management needs and requirements of the user.（符合系统管理数据的需要及用户的需求）
  • Guarantee that the data in the file are valid.
  • Optimize performance.（保证系统性能及用户对响应时间的要求）
  • Provide IO support for a variety of storage device types.(为多种存储设备类型提供IO支持)
  • Minimize or eliminate the potential for lost or destroyed data.（减少或避免数据丢失及损坏）
  • Provide a standardized set of I/O interface routines.（提供标准的I/O接口）
  • Provide I/O support for multiple users.（支持多用户环境下多个用户的I/O操作）

File Management Functions(文件管理功能)

  • Identify and locate a selected file.(识别并定位选定的文件)
  • Use a directory to describe the location of all files plus their attributes.(使用目录来描述所有文件及其属性的位置)
  • On a shared system describe user access control.(在共享系统上描述用户访问控制:权限)
  • Blocking for access to files.(把内存里面的记录组成块存储到磁盘。把磁盘里的块解开变成记录存到内存里面)
  • Allocate files to free blocks.（将文件分配给空闲块）
  • Manage free storage for available blocks.（管理可用块的空闲存储）

(文件系统和数据库系统：略)

pile（堆：一种文件结构），pipe（管道：拿来通信的）

Device Drivers(设备驱动)

  • Lowest level.（最低水平）
  • Communicates directly with peripheral devices.（直接与周边设备通讯）
  • Responsible for starting I/O operations on a device.（负责启动设备上的I/O操作）
  • Processes the completion of an I/O request.（处理I/O请求的完成）

Basic File System（基本文件系统）

  • Physical I/O（物理I/O）
  • Deals with exchanging blocks of data.（处理数据块的交换）
  • Concerned with the placement of blocks.（关心数据块的位置）
  • Concerned with buffering blocks in main memory.（关心缓冲块在主存中的位置）

Basic I/O Supervisor

  • Responsible for file I/O initiation and termination（负责文件I/O的启动和终止）
  • Control structures are maintained.（控制结构得到维护：磁盘分配表等）
  • Concerned with scheduling access to optimize performance.（关注调度访问以优化性能）
  • I/O buffers are assigned and secondary memory is allocated.（分配I/O缓冲区，并分配辅助内存）
  • Part of the operating system.（属于操作系统的一部分）

Logical I/O（逻辑I/O）

  • Enables users and applications to access records.（允许用户和应用程序访问记录）
  • Provides general-purpose record I/O capability（提供通用记录I/O功能）
  • Maintains basic data about file.（维护文件的基本数据）

Access Method（访问方法）

  • Reflect different file structures.（反映不同的文件结构）
  • Different ways to store and process data.（存储和处理数据的不同方式）
  • 如Pile、顺序访问、索引顺序访问、索引访问、Hash访问法等访问方法

File Organization and Access Methods（文件组织和访问方法）

File Organization（文件组织）

  • File organization refers to the logical structuring of the records as determined by the way in which they are accessed.（文件组织是指由访问记录的方式决定的记录的逻辑结构）
  • The physical organization of the file on secondary storage depends on the blocking strategy and the file allocation strategy.（辅助存储器上文件的物理组织取决于阻塞策略和文件分配策略）

Criteria for File Organization（文件组织标准）

  • Rapid access（快速访问）
    • Needed when accessing a single record.（访问单个记录时需要）
    • Not needed for batch mode.（批处理模式时不需要）
  • Ease of update（易于修改）
    • File on CD-ROM will not be updated, so this is not a concern.（文件位于CD-ROM上的不用考虑易修改）
  • Economy of storage（成本要低）
    • Should be minimum redundaney in the data.（冗余数据很少）
    • Redundaney can be used to speed access such as an index.（但是冗余数据可用于加速访问，如索引）
  • Simple maintenance（维护简单）
  • Reliability（可靠）

File Organization

  • Pile file（堆文件）
  • Sequential file（顺序文件）
  • Indexed sequential file（索引顺序文件）
  • Indexed file（索引文件）
  • Direct，or Hashed file（直接或hash文件）

Pile file（堆文件）

  • Data are collected in the order they arrive.（数据是按到达的顺序收集的）
  • Purpose is to accumulate a mass of data and save it.（目的是积累和保存大量的数据）
  • Records may have different fields.（记录可能有不同的域：结构很差）
  • No structure.（无结构）
  • Record access is by exhaustive search.（记录访问是通过穷举搜索实现的：查询很耗时间）

Sequential File（顺序文件）

  • Fixed format used for records.（固定格式用于记录）
  • Records are the same length.（记录的长度相同）
  • All fields the same (order and length).（所有域都是相同的(顺序和长度)）
  • Field names and lengths are attributes of the file.（域和长度是文件的属性）
  • One field is the key filed（一个域是关键域）
    • Uniquely identifies the record.（唯一标识记录）
    • Records are stored in key sequence.（记录按键顺序存储）
  • The only one that is easily stored on tapes as well as disk.（这是唯一一个可以轻松存储在磁带和磁盘上的）
  • Typically used in batch applications and the applications processing of all records.（通常用于批处理应用程序和处理所有记录的应用程序）
  • For querying and/or updating an individual record, it provides poor performance.（对于查询和/或更新单个记录，它提供了糟糕的性能）
  • New records are placed in a log file or transaction file.（新记录放在日志文件或事务文件中）
  • Batch update is performed to merge the log file with the master file.（执行批量更新以将日志文件与主文件合并:这样可以增加一定的查询速度）

Indexed Sequential File（索引顺序文件:就是多了个目录文件）

  • It maintains the key characteristic of the sequential file: records are organized in sequence based on a key field.（它保持了顺序文件的关键特征:记录是根据一个关键字段按顺序组织的）
    • Add an index to the file to support random access, and an overflow file.（在文件中添加一个支持随机访问的索引和一个溢出文件）
    • Index file contains key field and a pointer to the mainfile, and provides a lookup capability to quickly reach the vicinity of the desired record.（索引文件包含键字段和指向主文件的指针，并提供了快速查找所需记录的功能）
    • Overflow file is similar to the log file. It is integrated so that records in the overflow file are located by following a pointer from their processór record.（溢出文件类似于日志文件。它被集成在一起，这样就可以通过跟踪处理器记录中的指针来定位溢出文件中的记录。）
  • 索引顺序文件的每一条记录中都包含一个指向overflow文件的指针（该指针对应用程序透明）
    • New records are added to an overflow file
    • 如果新记录的前续记录在主文件（或overflow文件）中，则修改主文件（或overflow文件）中该前续记录指针，使之指向overflow文件中的新插入记录
    • The overflow is merged with the main file during a batch update.
    • Multiple indexes for the same key field can be set up to increiseefficiency.（可以为同一个键字段设置多个索引以提高效率）

例子

  • 若一级索引文件包含10，000条记录，再为之建立二级索引，其中包含100条记录
  • 在二级索引文件中平均检索50条记录，可以找到关键字
  • 再在一级索引文件中平均检索50条记录找到指向主文件的指针
  • 最后平均检索50条记录，找到目标记录，共计平均检索150条记录

Indexed File（索引文件）

  • 顺序文件和索引顺序文件都只允许按记录的唯一关键字检索记录，这不符合某些应用需要按多个字段检索记录的要求
  • 主文件的记录不必按关键字排序
  • Uses multiple indexes for different key fields.
  • May contain an exhaustive index（完备索引）that contains one entry for every record in the main file.
  • May contain a partial index（部分索引）that contains entries to records where the field of interest exists.

The Direct, or Hashed File（直接文件或哈希文件）

  • Directly access a block at a known address.
  • Key field required for each record.
  • 利用Hash函数，根据记录的关键字计算记录的存储位置，并按关键字访问记录，提高记录的访问效率
  • 常用于访问速率要求高、一次存取一条记录且记录为定长的文件.如文件目录、价格表、名单等文件记录的存取

File Directories（文件目录）

  • Contains information about files.（包含有关文件的信息。）
    • Attributes（属性）
    • Location（位置）
    • Ownership（所有权）
  • Directory itself is a file owned by the operating system.（目录本身是操作系统拥有的文件）
  • Provides mapping between file names and the files themselves.（提供文件名与文件本身之间的映射）
Structure for a Directory（目录的结构）
  • 可见，不同系统保存文件目录的方式不同，有的系统将萁中部分信息保存在文件头部，只将一些必要信息如文件名、文件大小、外存中的存储位置等保存在文件目
  • 对目录文伴的典型操作：
    • Search：按文件名检索文件目录录
    • Create file：建立文件时，必须为之建立一个新目录
    • Delete file：删除文件时，必须同时删除其文件目
    • List Directory：用户常需要列文件目录及文件的某些属性

从目录里面找到该文件的Index Node（Inode）（存储着文件的很多属性。）可以从inode得到文件的位置等等信息：这种方式更高效
还有一种是目录里面就存放了inode里面的信息，不需要额外的inode

B树的结构

File Sharing(文件分享)

  • In multiuser system，allow files to be shared among users.（在多用户系统中，允许用户之间共享文件）
  • Two issues
    • Access rights（访问权限）
    • Management of simultaneous access.（并行存取）（同时访问的管理。(并行存取)）

Access Rights(访问权限)

  • None
    • User may not know of the existence of the file.（用户可能不知道文件的存在）
    • User is not allowed to read the user direetory that includes the file.（不允许用户读取包含该文件的用户目录）
  • Knowledge（探知）
    • User can only determine that the file exists and who its owner is.（用户只能确定文件存在及其所有者是谁）
  • Execution（执行）
    • The user can load and execute a program but cannot copy it.（用户可以加载和执行一个程序，但不能复制它）
  • Reading
    • The user can read the file for any purpose, including copying and execution.（用户可以为任何目的读取文件，包括复制和执行）
  • Appending（添加）
    • The user can add data to the file but cannot modify or delete any of the file’s contents.（用户可以向文件中添加数据，但不能修改或删除文件的任何内容）
  • Updating
    • The user can modify, deleted, and add to the file’s data. This includes creating the file, rewriting it, and removing all or part of the data.（用户可以修改、删除和添加到文件的数据。这包括创建文件、重写文件以及删除所有或部分数据）
  • Changing protection （改变保护）
    • User can change access rights granted to other users.（用户可以更改授予其他用户的访问权限）
  • Deletion （删除）
    • User can delete the file.（用户可以删除文件）
  • Owners
    • Has all rights previously listed.（拥有以前列出的所有权利）
    • May grant rights to others using the ollowing classes of users.（可将权利授予其他使用该级别用户的用户：下面的……）
      • Specific user （特定的用户）
      • User groups （用户组）
      • All for public files（所有公共文件：公共用户）

Simultaneous Access（同时访问）

  • User may lock entire file when it is to be updated.（当要更新整个文件时，用户可以锁定它）
  • User may lock the individual records during the update.（用户可以在更新期间锁定个别记录）
  • Mutual exclusion and deadlock are issues for shared access（互斥和死锁是共享访问的问题）

Record Blocking记录组块方式

  • 记录是存取文件的逻辑单位，而数据块是I/O的基本单位，记录必须组织成数据块以便于I/O
  • 问题：
    • 数据块的长度固定还是可变？
    • 根据记录的平均长度如何设置数据块的长度？

数据块的长度固定还是可变？

大多数系统采用固定长度的数据块，以简化I/O操作、Buffer的分配及辅存中数据块的组织管理

如何设置数据块的长度？

  • 数据块越大，一次I/O传输的记录就越多
  • 大数据块适合文件顺序访问，因为这样可以减少I/O次数，加快处理速度
  • 当随机访问文件，或访问局部性很差时，传输的部分记录不会使用，效率低。
  • 另外，大数据块需要更大的I/O buffer，增加其管理复杂度

Fixed blocking（固定组块法）

  • 数据块由固定长度的若干条记录组成
  • 块内可能会存在一些被浪费空间

有内零头，不存在外零头

Variable Blocking：Spanned（可变长跨块组块法）

  • 数据块由变长记录组成，一条记录可以跨越两个数据块，可用指针记载一条跨块存储记录所在的下一数据块
  • 块内无浪费空间

没有内外零头

Variable Blocking Unspanned（可变长非跨块组块法）

  • 数据块由变长记录组成
  • 不允许一条记录跨越两个数据块
  • 块内可能存在被浪费空间

有内零头，不存在外零头

记录的大小小于块的大小

Secondary Storage Management外部存储管理

  • Space must be allocated to files.（必须为文件分配空间）
  • Must keep track of the space available for allocation.（必须跟踪（管理）可分配的空间）

Space Allocated to Files

  • When a new file is created, is the maximum space required for the file allocated at once?（当创建一个新文件时，一次分配文件所需的最大空间是多少?）
  • What size of portion should be used for file allocation?（文件分配应该使用多大的部分?）
  • What sort of data structure or table(FAT) is used to keep track of the portions assigned to a file?（使用哪种数据结构或表(FAT)来跟踪分配给文件的部分?）

Preallocation（预分配策略：一种固定分配策略，以后不再改变文件的大小）

  • Need the maximum size for the file at the time of creation.（在创建时需要文件的最大大小）
  • Difficult to reliably estimate the maximum potential size of the file.（难以可靠地估计文件的最大潜在大小）
  • Tend to overestimated file size so as not to run out of space.（倾向于高估文件大小，以免耗尽空间）

Dynamic Allocation（动态分配）

  • Allocates space to a file in portions as needed.

Portion Size（分区大小）

  • 两个极端情况：
    • 足够大：能容纳整个文件
    • 最小：1个磁盘块大小
  • Tradeoff：
    • 单个文件的效率vs整个系统的效率

Portion Size: Trade-off

  • Contiguity of space increase performance, especially for Retrieve_Next operations, and greatly for transactions running in a transaction-oriented operating system.
  • Having a large number of small portions increase the size of tables needed to manage the allocation information.
  • Having fixed-size portions (for example, blocks)simplifies the reallocation of space.
  • Having variable-size or small fixed-size portions minimizes waste of unused storage due to overallocation.

Methods of File Allocation（分配方法）

  • Contiguous allocation（连续分配）
  • Chained allocation（链接分配）
  • Indexed allocation（索引分配）

Contiguous allocation（连续分配）

  • Preallocatin strategy using variable-size portions.
  • Single set of blocks is allocated to a file at the time of creation.
  • Only a single entry in the file allocation table.
    • Starting block and length of the file.

Allocation Strategies（分配策略：可变分区）

  • First fit：Choose the first unused contiguous group of blocks of sufficient size.
  • Best fir：Choose the smallest unused group of sufficient size.（内零头最小）
  • Nearest fir：Choose the unused group of sufficient size that is closest to the previous allocation for the file to increase locality.

Contiguous Allocation （连续的分配）

  • External fragmentation will occur.（外零头存在）
  • It is necessary to perform a compaction algorithm to free up additional space on the disk.（必须执行压缩算法来释放磁盘上的额外空间）

Chained Allocation（链接分配：没有外零头**）

  • Allocation on basis of individual block.
  • Each block contains a pointer to the next block in the chain.
  • Only single entry in the file allocation table.
    • Starting block and length of file
  • No external fragmentation.
  • Best for sequential files.
  • No accommodation of the principle of locality.

Chained Allocation（链接分配）

  • To select an individual block of a file requires tracing trough the chain to the desired block.（要选择文件中的一个块，需要通过链跟踪到所需的块）
  • To access many blocks at a time, a series of accesses to different parts of the disk are required.（为了一次访问多个块，需要对磁盘的不同部分进行一系列访问）
  • To overcome this problem, some system periodically consolidate files.（为了克服这个问题，一些系统定期合并文件）

Indexed Allocation（索引分配）

  • File allocation table contains a separate one-level index for each file.（一级索引）
  • The index has one entry for each portion allocated to the file.
  • The file index for a file is kept in a separate block，and the file allocation table contains block number for the index.
  • Allocation may be on the basis of fixed-size blocks.
  • Or allocation may be on the basis of variable-size portions.
    Indexed Allocation
  • Eliminates external fragmentation.
  • File consolidation may be done from timeto time, reducing the size of the index in the case of variable-size portions.
  • Supports both sequential and direct access to the file, and thus is the most popular form of file allocation.

多级索引文件

Free Space Management（空闲空间管理）

Disk Allocation Table：

  • Bit table（位表）
  • Chained free portion（空闲分区链）
  • Indexing（索引）

Bit Table

  • 位表，又称为位示图，其中的每一位对应一个磁盘块。位的值为0或1，分别表示磁盘块空闲，或磁盘块已分配
  • 利用位表容易找到一个或一组空闲盘块
  • 位表适合于以上各种文件分配方法
  • 位表很小，可以装入内存

Chained Free Portion

  • 每个空闲分区包含一个指向下一个分区的指针，并记载分区大小
  • 无空闲分区表空间开销
  • 适合于各种文件分配方法
  • 若每次分配一个磁盘块，则可取空闲分区链的第一个盘块进行分配，并调整空闲分区链首指针和分区链大小
  • 若采用可变分区法，可用首次适应算法，从链表头开始查找，找到的第一个适合的分区则可分配，然后调整空闲分区链首指针和分区链大小

Indexing

  • 将空闲分区视为文件，按文件存储空间分配法为空闲分区建立索引
  • 索引表中为每一个空闲分区建立一个索引项
  • 为可变分区建立索引比为磁盘块建立索引效率高
  • 适合于各种文件分配法
