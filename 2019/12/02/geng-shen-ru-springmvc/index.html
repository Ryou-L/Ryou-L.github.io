纯Java配置开发

无xml文件到开发

public class MyWebApplicationInitializer implements WebApplicationInitializer {

    @Override
    public void onStartup(ServletContext servletCxt) {

        // Load Spring web application configuration
        //基于注解开发，需要注解上下文的初始化
        AnnotationConfigWebApplicationContext ac = new AnnotationConfigWebApplicationContext();
        //通过调用register方法法册配置类
        ac.register(AppConfig.class);
        //刷新
        ac.refresh();

        // Create and register the DispatcherServlet
        //前端控制器的注册
        DispatcherServlet servlet = new DispatcherServlet(ac);

        //注册映射规则
        ServletRegistration.Dynamic registration = servletCxt.addServlet("app", servlet);

        registration.setLoadOnStartup(1);

        //url-pattern/
        registration.addMapping("/app/*");
    }
}

异步请求

从spring3.0开始，要做异步请求处理非常简单，只需要添加一个注解 @Async即可。

异步请求VS同步请求
同步请求阻塞客户端，直到操作完成。在这种情况下，浏览器的javascript引擎被阻止。

在servlet.xml文件里面添加

<bean class="org.springframework.web.multipart.commons.CommonsMultipartResolver" id="multipartResolver">
    <property name="maxUploadSize" value="1024"/> <!--文件上传的大小-->
</bean>

Controller层

@Controller
public class FileUpLoad {

    @PostMapping("fileupload")
    public String fileupload(@RequestParam("file")MultipartFile multipartFile){
        System.out.println(multipartFile.getOriginalFilename());
        try {
            System.out.println(multipartFile.getBytes());
        }catch (Exception e){
            e.printStackTrace();
        }
        return "success";
    }
}

<form action="${ctx}/fileupload" method="post" enctype="multipart/form-data">

</form>

文件下载

使用Apache工具类IOUtil可以快速完成文件下载功能.
此外需要对响应设置响应头控制下载

response.setHeader("content-disposition","attachment;filename="+file.getName());

如果是中文,有的浏览器有乱码,以chrome为例,重新构建文件名即可

response.setHeader("content-disposition","attachment;filename="+new String(file.getName().getBytes("UTF-8"),"ISO-8859-1"));

异常解析器

在spring里面定义了一个接口HandlerExceptionResolver@since 22.11.2003，我们以 simpleMappingExceptionResolver 为例说明异常解析器其有多个属性配置，重点的我们说下

其有多个属性配置，重点的我们说下
exceptionMappings 描述哪些异常映射比如

<property name="exceptionMappings">
    <props>
        <prop key="java.lang.RuntimeException">exp</prop>
    </props>
</property>

通过一组属性文件描述具体的异常如何处理。

  • defaultStatusCode指定默认的响应状态码比如 404 500
  • defaultErrorView 指定默认的错误视图如果在exceptionMappings已经定义过了，那么优先使用exceptionMappings里面的定义的视图，如果没有则使用defaultErrorView
